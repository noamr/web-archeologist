# Web Standards Archeologist

A specialized tool for standards engineers to trace spec prose back to its historical "Why". It bridges the gap between modern specification fragments and the 20-year history of GitHub issues, Bugzilla reports, and W3C mailing list discussions.

## The Problem
Specification source files (like WHATWG's `html/source`) are massive (130,000+ lines). Standard git blame often only shows the last editorial cleanup or infrastructure update, hiding the actual normative rationale that might date back to 2006.

## The Solution: The "Chain of Evidence"
This tool automates the archeological protocol:
1. **URL/Fragment -> Source Line**: Uses heuristics to find where a `#fragment` is actually defined in the source (even if generated by Bikeshed/Wattsi).
2. **Deep Blame**: Uses `git log -L` to trace the history of a specific line through refactors and moves.
3. **Link Extraction**: Identifies and categorizes references to:
   - **GitHub**: PRs and Issues.
   - **W3C Bugzilla**: Historical bug reports.
   - **SVN Revisions**: Direct search links for `lists.w3.org` archives.
   - **Browser Bugs**: Chromium, WebKit, and Gecko trackers.

## Installation
```bash
git clone https://github.com/noamr/web-archeologist.git
cd web-archeologist
```

## Usage

### 1. Trace a Spec Section
Pass a WHATWG or W3C URL. The tool will automatically clone/update the required repo in `~/.gemini/cache/specs`.
```bash
python3 archeologist.py "https://html.spec.whatwg.org/#the-insertion-mode"
```

### 2. Trace a GitHub Issue
Find all commits and related discussions associated with a specific issue.
```bash
python3 archeologist.py "whatwg/html#5229"
```

### 3. Get JSON Output
For integration with other tools or LLM agents:
```bash
python3 archeologist.py --json "#foster-parenting"
```

## Supported Standards
- **HTML**: `whatwg/html`
- **CSS**: `w3c/csswg-drafts` (All modules)
- **DOM**: `whatwg/dom`
- **Fetch**: `whatwg/fetch`
- **URL, XHR, Storage, Infrastructure, etc.**
